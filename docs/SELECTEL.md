# ТЗ: запуск на Selectel (GPU-сервер) с минимизацией трат

Этот сервер = **GEN** (наш проект + ComfyUI на GPU). Твой сайт (**SRC**) шлёт сюда
запросы. Ниже — что именно брать, как настроить и как не сжечь деньги.

---

## 0. Главное про деньги (читать первым)

- На скрине выбран **месячный** тариф (39 421 ₽/мес) — НЕ бери его для старта.
- Переключи цену на **«Час»**. GPU — это ~90% стоимости, платишь, пока сервер
  ВКЛЮЧЁН. Правило: **включил → сгенерил/потестил → выключил/удалил.**
- Баланс 676 ₽ ≈ ~12 часов T4. Для реальной работы — пополнить и работать
  почасово, а не держать 24/7 на старте.
- 4 рычага экономии (по убыванию эффекта):
  1. **Почасовая оплата + останавливать/удалять** простаивающий сервер.
  2. **Прерываемый (spot) сервер** — дешевле, но живёт ≤24 ч и может быть убит.
     Идеален для тестов/пакетной генерации, НЕ для 24/7-прода.
  3. **Правильная (не избыточная) конфигурация**: 8 vCPU / 32 ГБ хватает,
     не бери 192 ГБ RAM и лишние ядра.
  4. **Turbo-модели / меньше шагов** (см. OPTIMIZATION.md) → меньше GPU-секунд
     на картинку → меньше денег.

---

## 1. Какую видеокарту брать

| Задача | Рекомендация | Почему |
|---|---|---|
| **Старт/тест, фото-лицо, бюджет** | **A2 16 ГБ** или **Tesla T4 16 ГБ** | дешевле всего, SDXL+InstantID влезает (fp8) |
| **Баланс: хорошее фото + лёгкое видео** | **L4 24 ГБ** ⭐ или **A5000 24 ГБ** | современные, 24 ГБ под FLUX/InstantID и i2v |
| **Серьёзное видео/анимация** | **A100 40 ГБ** или **A30 24 ГБ** | тяжёлые i2v-модели, быстрее в 2–3× |

Рекомендация для тебя: **начни с L4 24 ГБ, 8 vCPU, 32 ГБ RAM** (золотая середина
цена/возможности). Если сейчас только фото и совсем экономишь — **A2 16 ГБ /
T4 16 ГБ**. A100 — подключай точечно под видео-пакеты.

> Локальный NVMe-диск (быстрее) доступен только с A100 40 ГБ, A30, RTX 6000 Pro.
> Для L4/T4/A2 — обычный SSD, это нормально.

---

## 2. Операционная система (образ)

Бери **GPU-optimized с Docker**:
> **Ubuntu 24.04 LTS 64-bit GPU Driver 590 Open — Docker**

Почему: уже стоят NVIDIA-драйвер + Docker + nvidia-container-toolkit — не нужно
ничего ставить вручную, наш `docker compose` сразу видит GPU.

---

## 3. Диск

- **НЕ 20 ГБ** — модели большие (FLUX ~24 ГБ, SDXL ~7 ГБ, видео-модели 10–40 ГБ,
  InstantID/antelopev2, ComfyUI, образы Docker).
- Бери **100–150 ГБ SSD** (диск дёшев относительно GPU — экономить тут не на чем).
- Результаты генерации храни в **S3 (Selectel Object Storage)**, а не на диске —
  тогда диск не разрастается. У нас `STORAGE_PROVIDER=s3` это поддерживает.

---

## 4. Сеть и безопасность

- **Публичный IP** — да (чтобы SRC слал запросы из интернета). Если SRC тоже на
  Selectel — можно через приватную подсеть (бесплатнее и безопаснее).
- **Группы безопасности (фаервол)** — открыть только:
  - `22/tcp` (SSH) — только с твоего IP;
  - `443/tcp` (HTTPS API) — только с IP сервера SRC (allowlist);
  - **НЕ открывать** `8188` (ComfyUI) и `8000` напрямую в интернет.
- **SSH-ключ** — обязательно (как требует образ). Пароль root сохрани, но вход по
  ключу.

---

## 5. Конфигурация, которая НЕ нужна (режь смело)

- Месячный тариф → **Час**.
- 64–192 ГБ RAM → достаточно **32 ГБ**.
- Много vCPU → **8 vCPU**.
- 2×/4× GPU → старт на **1× GPU**.
- Включи **Прерываемый сервер** на время тестов/разработки.

---

## 6. Пошаговый запуск (после создания сервера)

```bash
ssh root@<публичный-IP>            # по SSH-ключу
git clone <repo> ai-generation-api && cd ai-generation-api
cp .env.example .env
nano .env    # выставить:
#   GENERATION_PROVIDER=comfyui
#   COMFYUI_URL=http://comfyui:8188   (или http://localhost:8188)
#   SAFETY_PROVIDER=insightface       # реальная проверка 21+
#   SAFETY_MIN_AGE=21
#   STORAGE_PROVIDER=s3 + S3_* (Selectel Object Storage)
#   WEBHOOK_SIGNING_SECRET / INTERNAL_JWT_SECRET — сменить на свои

# поднять наш стек (api, worker, postgres, redis, миграции):
docker compose up -d --build

# ComfyUI на GPU: поставить рядом (свой контейнер/образ с моделями),
# загрузить чекпоинты (FLUX/SDXL) и ноды InstantID/PuLID, открыть :8188 локально.

# выдать API-ключ для SRC-сервера:
docker compose exec api python -m scripts.create_api_key "Site" --callback https://src/api/gen-callback

# HTTPS наружу: nginx/traefik + сертификат на :443 → проксирует на api:8000
```

Проверка: `curl https://<домен>/health` → `{"status":"ok","provider":"comfyui"}`.

Полные детали интеграции/безопасности — [TZ.md](TZ.md), [INTEGRATION.md](INTEGRATION.md).

## 7. Как писать промты (после запуска)

Промты — это правка YAML-файлов режимов + проверка без GPU:
```bash
# отредактировал config/modes/photo/PHOTO_MODE_1.yaml (prompt_template/params)
curl -X POST https://<домен>/modes/PHOTO_MODE_1/preview \
  -H "X-API-Key: <key>" -H "Content-Type: application/json" \
  -d '{"metadata":{"subject":"woman"}}'        # увидишь итоговый промт
# применить без рестарта:
TOKEN=$(docker compose exec -T api python -m scripts.mint_internal_token --subject ops)
curl -X POST https://<домен>/admin/modes/reload -H "Authorization: Bearer $TOKEN"
```
Подробно (переменные, плейсхолдеры, качество) — [PROMPTING.md](PROMPTING.md) и
раздел 6–7 в [TZ.md](TZ.md).

## 8. Сценарии стоимости (ориентир, почасово)

Точные цены смотри в Selectel при переключении на «Час» (для L4/A5000/A100 они
выше T4). Логика экономии одна:

| Режим работы | Как платишь | Комментарий |
|---|---|---|
| **Разработка/тест** | spot + почасово, выключать | минимально; T4/A2/L4 |
| **MVP, мало трафика** | почасово, поднимать на активные часы | дёшево, но не 24/7 |
| **Прод 24/7** | обычный (не spot) сервер | дороже; считай по CAPACITY.md |

Снизить себестоимость в проде: turbo-LoRA/малые шаги, тёплый GPU, отдельный
дешёвый CPU-сервер под API/БД + GPU-сервер только под генерацию (можно
останавливать в простое), результаты в S3.

## 9. Что выбрать прямо сейчас (итог)

- **GPU:** L4 24 ГБ (баланс) или T4/A2 16 ГБ (макс. экономия, фото).
- **CPU/RAM:** 8 vCPU / 32 ГБ.
- **ОС:** Ubuntu 24.04 LTS GPU Driver 590 Open — **Docker**.
- **Диск:** 100–150 ГБ SSD.
- **Тариф:** **Час**, на тесты — **Прерываемый**.
- **Сеть:** публичный IP + фаервол (22 c твоего IP, 443 c IP SRC, 8188/8000 закрыть).
- **После старта:** `.env` (comfyui + insightface + s3) → `docker compose up -d` →
  поставить ComfyUI с моделями → выдать API-ключ → писать промты через `/preview`.

⚠️ Сейчас баланс 676 ₽ — хватит только на короткий тест. Переключись на «Час»,
подними **прерываемый T4/A2/L4**, протестируй и **сразу выключи/удали**, иначе
деньги уйдут за часы.

## 10. Дев-сервер для промтинга (одноразовый, максимально дёшево)

Цель: дешёвый сервер, чтобы разрабатывать промты/режимы и собрать ComfyUI-
workflow. Потом пересядешь на нормальный, этот — удалишь.

### Главное: для САМИХ промтов GPU почти не нужен
Написание/проверка промтов идёт через `POST /modes/{id}/preview` и mock-движок —
это **CPU**, GPU не задействован. То есть львиную долю работы (текст промтов,
переменные, режимы, интеграция, callbacks) можно делать на **дешёвой машине без
мощного GPU** или вообще на `GENERATION_PROVIDER=mock`.

GPU нужен только чтобы **увидеть реальную картинку** и отладить ComfyUI-workflow.

### Что брать под дев (по бюджету)
| Вариант | Цена | Для чего годится |
|---|---|---|
| **A2000 6 ГБ** (твой пример, ~42 ₽/ч) | дёшево | пайплайн, промты, mock, учить ComfyUI на SD1.5/мелких моделях. ⚠️ 6 ГБ — FLUX/SDXL+InstantID/видео НЕ потянет |
| **T4 16 ГБ / A2 16 ГБ** (~54 ₽/ч) | чуть дороже | реально гонять SDXL+InstantID, отлаживать фото-промты |
| **L4 24 ГБ** | дороже | как будущий прод (фото+лёгкое видео), проверить «как будет» |

### Рекомендация
- Если сейчас задача — **писать промты и собрать структуру/workflow** и поучиться:
  **A2000 6 ГБ за ~42 ₽/ч** отлично подходит (дёшево). Но помни про лимит 6 ГБ —
  тяжёлые модели не запустятся.
- Если нужно **видеть реальное фото-лицо** (InstantID/SDXL): бери **T4/A2 16 ГБ**.
- Всегда: **тариф «Час» + Прерываемый сервер + удалить после сессии.**

### Настройки дев-сервера (как на твоём скрине)
- **ОС:** Ubuntu 24.04 LTS GPU Driver 590 Open — **Docker**.
- **Диск:** 120 ГБ (ок; хватит под модели и Docker).
- **CPU/RAM:** 12 vCPU / 32 ГБ (можно меньше для экономии — узкое место не CPU).
- **Сеть:** публичный IP (чтобы ходить по SSH и дёргать API).
- **Прерываемый сервер:** включить (дешевле, для одноразового дева идеально).
- **SSH-ключ:** добавить.

### Запуск дев-сервера (через GitHub — см. GITHUB.md)
```bash
ssh root@<IP>
git clone https://github.com/<логин>/ai-generation-api.git && cd ai-generation-api
cp .env.example .env          # для промтинга можно оставить GENERATION_PROVIDER=mock
docker compose up -d --build
# писать/проверять промты:
docker compose exec api python -m scripts.create_api_key "Dev"
# POST /modes/{id}/preview ... правишь YAML ... /admin/modes/reload
```
Когда закончил — **удали сервер** в панели Selectel (почасовая оплата остановится).

> Дешевле всего: промты можно отрабатывать вообще на mock (без GPU), а GPU
> поднимать точечно, только когда нужно увидеть реальный результат.
