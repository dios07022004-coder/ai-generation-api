# Оптимизация времени генерации

Честно: **главное время — это инференс модели на GPU**, его задаёт не код API, а
workflow/модель/железо. Код API уже оптимизирован (см. §3). Ниже — рычаги по
убыванию эффекта.

## 1. Самые большие выигрыши (workflow/модель, ComfyUI)

| Приём | Эффект | Где |
|---|---|---|
| **Distilled/turbo модели и LoRA** (SDXL-Lightning, LCM, Hyper-SD, DMD2; для видео — LTX-Video, CausVid) | шаги 30→**4–8**, x3–x8 быстрее | выбор модели в `models.yaml` + workflow |
| **Меньше шагов (steps)** при том же качестве с turbo-LoRA | линейно к шагам | `params.steps` в режиме |
| **fp8/fp16 веса** вместо fp32 | x1.5–2, меньше VRAM | загрузчик модели в workflow |
| **TensorRT / torch.compile** компиляция UNet | x1.3–2 | ноды TensorRT в ComfyUI |
| **SageAttention / FlashAttention / xformers** | x1.2–1.5 | флаги запуска ComfyUI |
| **Генерация в меньшем разрешении + апскейл** (4x-UltraSharp/RealESRGAN) | x2–4 на «тяжёлых» пикселях | узлы upscale в workflow |
| **Видео: меньше кадров + интерполяция** (RIFE/FILM удваивает fps) | x1.5–2 | workflow |

## 2. Держать GPU «тёплым» (убрать холодный старт)

- ComfyUI должен **держать модель в VRAM между задачами** (не перезагружать).
  Запуск с `--highvram`/`--gpu-only`, один постоянный процесс ComfyUI.
- **Колокация**: воркер Celery и ComfyUI на одной ноде → нет сетевого хопа за
  результатом. `COMFYUI_URL=http://localhost:8188`.
- Прогрев: после старта прогнать 1 «пустую» генерацию, чтобы модель/компиляция
  уже были в памяти к первому пользователю.

## 3. Что уже сделано на уровне кода API

- **Быстрый опрос готовности** ComfyUI: `COMFYUI_POLL_INTERVAL=0.5` (было 1.0) —
  короткие фото-задачи забираются вдвое быстрее. Настраивается в `.env`.
- **Раздельные очереди** `generation`/`generation_video` — видео не задерживает фото.
- **Один общий загрузчик изображений** (без лишних зависимостей/повторных качаний).
- **`prefetch_multiplier=1`, `concurrency=1`** — одна тяжёлая задача на GPU-воркер
  (правильно для GPU; повышать только при достаточной VRAM под параллельность).
- **Safety-проверка не качает фото**, если провайдер `none/mock` (ноль оверхеда);
  на `insightface` добавляет ~50–150 мс на лицо.

## 4. Пропускная способность (а не только скорость одной задачи)

- Несколько GPU-воркеров + **KEDA-автоскейл по длине очереди** (см. k8s/).
- На картах с большим VRAM иногда выгодно 2 параллельных потока на GPU
  (`concurrency=2`) — мерить, не наугад.
- Хранилище **R2/S3** + CDN — быстрая отдача результата и загрузка исходников.

## 5. Практический рецепт «быстро и качественно»

- Фото: SDXL/Flux + **turbo-LoRA (6–8 шагов)** + InstantID/PuLID + лёгкий
  face-detailer + апскейл 1.5×. Цель ~2–4 с на 4090.
- Видео: **LTX-Video** (быстрый) или distilled i2v, 2–3 с клип, fps через RIFE.
  Цель ~30–60 с на A100.
- Всегда фиксируй `seed` для воспроизводимости и меряй p95 `generation_seconds`
  (Grafana), а не «на глаз».

> Точные цифры зависят от карты — прогони одну реальную генерацию и подставь G в
> [CAPACITY.md](CAPACITY.md), чтобы пересчитать пропускную способность.
