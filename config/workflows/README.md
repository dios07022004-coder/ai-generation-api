# ComfyUI Workflows

Сюда кладутся workflow, экспортированные из ComfyUI кнопкой
**Save (API Format)** — файл `<имя>.json`. В режиме указывается `workflow: <имя>`.

## Плейсхолдеры

В JSON-шаблоне в значениях нод пиши плейсхолдеры — провайдер заменит их
перед отправкой в ComfyUI:

| Плейсхолдер | Значение |
|---|---|
| `{{prompt}}` | отрендеренный позитивный промт режима |
| `{{negative}}` | негативный промт режима |
| `{{image_url}}` | URL основного референса (лицо/персонаж) |
| `{{reference_0}}`, `{{reference_1}}`, … | доп. референсы (мульти-персонаж) |
| `{{reference_count}}` | число доп. референсов |
| `{{driving_url}}` | управляющее видео/поза (движения) |
| `{{seed}}` | seed из params (или 0) |
| `{{reference_strength}}` | сила влияния референса |
| `{{param.<key>}}` | любой ключ из `params` режима (напр. `{{param.width}}`) |
| `{{model.name}}` | имя модели из models.yaml |

## Сохранение лица

Логика сохранения лица — целиком внутри workflow (ноды **InstantID / IP-Adapter /
PuLID** + загрузчик исходного фото по `{{image_url}}`). В коде её нет.
Пример-заглушка: `photo_instantid.json` (замени на свой реальный экспорт).
